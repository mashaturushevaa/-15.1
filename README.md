# Практична робота 15.1: Deployment Automation (GitOps Approach)
**Виконала:** Марія Турушева  
**Тема:** Автоматизація розгортання конфігурації OpenHAB з використанням GitHub Actions.

## 1. Концепція проєкту
У цій роботі реалізовано сучасний DevOps-підхід — **GitOps**. Основна ідея полягає в тому, що репозиторій GitHub є єдиним джерелом істини для налаштувань системи. Будь-яка зміна в коді автоматично запускає процес перевірки та оновлення системи без ручного втручання.

## 2. Стек технологій
* **GitHub Actions:** для створення автоматизованих сценаріїв (Pipelines).
* **YAML:** для опису логіки автоматизації.
* **Git:** для керування версіями конфігурації.

## 3. Опис автоматизації (CI/CD Pipeline)
У репозиторії налаштовано файл сценарію `.github/workflows/deploy.yml`, який автоматично запускається при кожній події `push`. "Робот" виконує наступні кроки:
1. **Checkout:** завантажує актуальний код конфігурації.
2. **Validate:** проводить симуляцію перевірки синтаксису файлів (Items, Rules). Це гарантує, що в систему не потрапить помилковий код.
3. **Deploy:** імітує процес синхронізації та розгортання нової версії на Edge-вузлі.

## 4. Результати
* **Автоматизація:** повний цикл від зміни файлу до оновлення системи відбувається автоматично.
* **Надійність:** виключено людський фактор при перенесенні налаштувань.
* **Версіонування:** кожна зміна зберігається в історії, що дозволяє миттєво повернутися (Rollback) до стабільної версії у разі збою.
